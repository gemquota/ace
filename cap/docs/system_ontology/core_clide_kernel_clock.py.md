# core_clide_kernel_clock.py

## 🧩 Metadata
- **Module:** `CLIDE`
- **Subsystem:** `Kernel`
- **Function:** Lamport logical clock for event ordering in a distributed system.
- **Source Path:** `core/clide/kernel/clock.py`

## 📝 Description
`core_clide_kernel_clock.py` provides a implementation of a Lamport logical clock to establish partial ordering of events across a potentially distributed system. It ensures that events have consistent and monotonically increasing timestamps, which is critical for causal reasoning and state synchronization in the CAP system.

## 🛠️ Technical Reality

### 1. LogicalClock Implementation
The core class `LogicalClock` maintains a simple internal counter representing the logical time.
-   **Initialization:** Starts the clock at a configurable `start_time` (default: 0).
-   **Tick Operation:** Increments the internal counter and returns the new value. This is used for generating timestamps for locally generated events.
-   **Update Operation:** Synchronizes the clock with a received logical time. It sets the internal counter to the maximum of its current value and the received value, then increments it by one. This ensures that a received event's causal parents always have lower logical timestamps than any events subsequently generated locally.

### 2. Singleton Interface
The module provides a global singleton instance of the `LogicalClock` for the local kernel, along with helper functions for ease of use:
-   **`get_next_logical_time()`:** Increments and returns the next logical timestamp from the global clock.
-   **`update_clock(received_time)`:** Synchronizes the global clock with an external timestamp.
-   **`get_real_timestamp()`:** Provides a convenience function to get the current real-world Unix timestamp.

### 3. Usage in Event Sourcing
The logical clock is a foundational component of the `Event` structure. Every event generated by CLIDE includes a `logical_clock` field, which is used by the `PIE` engine to reconstruct causal chains and verify the temporal integrity of trace data.

## 🔗 Dependencies
-   **Python standard library:** `time` for real-world timestamps.
-   **CLIDE Event Kernel:** Consumes logical timestamps for event creation.
