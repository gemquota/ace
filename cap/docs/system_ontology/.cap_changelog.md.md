#### The SemVer Logic Breakdown:
*   **Major Version (X):** Incremented when the tri-pillar core (`core/clide`, `core/apc`, `core/pie`) undergoes a fundamental API-breaking change (e.g., changing the event schema in `cap_events.db`).
*   **Minor Version (Y):** Incremented when new capabilities (e.g., new PIE flavours like `introspection.py`, new mesh protocols) are added without breaking backward compatibility.
*   **Patch Version (Z):** Incremented for bug fixes, documentation updates, and "slop" removal. This is the most frequent mutation point for the PIE Sweeper Agent.
